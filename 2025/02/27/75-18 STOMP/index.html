75-18 STOMP

STOMP (Simple Text Oriented Messaging Protocol) 详解

STOMP（Simple/Streaming Text Oriented Messaging Protocol）是一种简单的基于文本的消息协议，常用于 WebSocket 和消息代理（如 ActiveMQ, RabbitMQ）之间的通信。它提供了一种 标准化的消息格式和交互方式，使客户端和服务器可以使用 发布/订阅（Pub/Sub）模式进行通信。

你可以把 WebSocket 想象成一条电话线路，它可以让前后端保持连接并实时通信。但如果所有消息都直接在这条线路上传输，那管理起来就会变得非常混乱：

  • 谁该收到哪些消息？
  • 消息格式怎么解析？
  • 怎么确保消息是正确的？
  • 用户如何身份认证？

而 STOMP 就是一个“电话助理”，它提供了一套规范化的通信方式，让消息可以被高效地管理。

STOMP 到底做了什么？

STOMP 主要做了 3 件事情：

  1. 定义了一套标准消息格式
    • WebSocket 只能发送原始数据，STOMP 定义了统一的消息格式（就像 HTTP 头部一样）。
    • 它的消息格式包括：
      • 命令（Command）：比如 SEND（发送消息）、SUBSCRIBE（订阅频道）。
      • 头部（Headers）：存储元数据，比如 destination:/topic/chat（表示发给哪个频道）。
      • 消息体（Body）：真正的消息内容。
    • 这样，每个消息的结构都是清晰的，前后端都能轻松解析，而不需要自己设计一套协议。
  2. 支持“发布/订阅”模型
    • 传统 WebSocket 只能在客户端和服务器之间一对一传输数据。
    • 但 STOMP 允许多个客户端订阅同一个“频道”（就像聊天室里的所有人）。
    • 服务器只需要把消息发送到某个频道，所有订阅这个频道的用户都会自动收到。
    • 这解决了多人通信的问题，比如聊天室、股票推送等。
  3. 提供身份认证和扩展性
    • 由于 STOMP 的消息头部可以携带额外的信息，比如 Authorization 头部，后端可以用它进行身份验证。
    • STOMP 也可以与**消息代理（Message Broker）**配合，比如 RabbitMQ、Kafka，让消息更稳定和可扩展。

1. 为什么需要 STOMP？

WebSocket 本身只是一个全双工通信协议，它允许客户端和服务器直接发送和接收数据。但是：

  • WebSocket 没有内置的消息格式 → 需要自己设计数据结构。
  • WebSocket 没有提供订阅功能 → 需要手动管理谁订阅了哪些消息。
  • WebSocket 没有身份认证 → 需要自己实现 Token 或其他安全机制。

STOMP 解决了这些问题，使得 WebSocket 更加易用和功能强大。

2. STOMP 的核心概念

STOMP 是一种基于 帧（Frame） 的协议，每个请求或响应都是一个 帧，类似于 HTTP 报文，格式如下：

  • COMMAND：操作类型，如 SEND、SUBSCRIBE、CONNECT 等。
  • header：键值对的消息头，提供额外的消息信息。
  • message-body：实际传输的数据内容（可以是 JSON、XML、文本等）。
  • ^@（\0）：STOMP 消息的结束标志。

常见的 STOMP 命令

命令 作用
CONNECT 客户端连接到服务器
SUBSCRIBE 订阅一个目的地（类似 RabbitMQ 主题）
UNSUBSCRIBE 取消订阅
SEND 发送消息到指定目的地
MESSAGE 服务器向客户端发送的消息
DISCONNECT 断开连接

7. STOMP 的优势

  • 简单易用：基于文本格式，类似 HTTP，非常容易解析。
  • 支持消息订阅：内置 订阅/广播 机制（WebSocket 本身不支持）。
  • 支持消息队列：可以 结合 RabbitMQ、ActiveMQ 进行分布式通信。
  • 支持身份认证：可以 携带 Token 进行认证。

8. 总结

问题 WebSocket STOMP
数据格式 纯文本/二进制 结构化的消息帧
订阅机制 需要手动管理 内置 SUBSCRIBE 机制
身份认证 需要自定义 可以携带 Token
扩展性 适合点对点通信 适合广播、群聊、分布式系统

学习重点

  1. STOMP 是 WebSocket 的增强协议，解决了 WebSocket 没有消息格式、订阅功能、认证机制的问题。
  2. Spring Boot 原生支持 STOMP，可以结合 WebSocket 实现实时聊天、消息推送。
  3. 结合 STOMP 可以使用 RabbitMQ、ActiveMQ，实现 更强大的分布式消息系统。
