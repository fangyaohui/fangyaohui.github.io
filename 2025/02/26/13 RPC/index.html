要想了解什么是gRPC，那么我们可以先了解一下什么是RPC。

什么是RPC

RPC（Remote Procedure Call Protocol）远程过程调用协议，目标就是让远程服务调用更加简单、透明。

RPC可以理解成为了实现两个进程之间的通信。

为什么需要用RPC

当业务越来越多，应用也会越来越多，这时候需要将公共的业务逻辑抽离出来，形成一个独立的service服务，所以这时候需要一种能够在不同微服务进行通信或者说不同进程之间进行通信的方式，这就是RPC。

RPC和HTTP区别

　　　　RPC 和 HTTP都是微服务间通信较为常用的方案之一，其实RPC 和 HTTP 并不完全是同一个层次的概念，它们之间还是有所区别的。

　　　　 1. RPC 是远程过程调用，其调用协议通常包括序列化协议和传输协议。序列化协议有基于纯文本的 XML 和 JSON、二进制编码的Protobuf和Hessian。传输协议是指其底层网络传输所使用的协议，比如 TCP、HTTP。

　　　　 2. 可以看出HTTP是RPC的传输协议的一个可选方案，比如说 gRPC 的网络传输协议就是 HTTP。HTTP 既可以和 RPC 一样作为服务间通信的解决方案，也可以作为 RPC 中通信层的传输协议（此时与之对比的是 TCP 协议）。

RPC常用框架

  1. gRPC： 由Google开发的高性能、跨语言的RPC框架，基于HTTP/2和Protocol Buffers，支持多种编程语言，包括C、C++、Java、Go、Python等。
  2. Apache Thrift： 由Facebook开发的跨语言的RPC框架，支持多种编程语言，包括C++、Java、Python、PHP等，提供了丰富的数据类型和序列化协议支持。
  3. Apache Dubbo： 由阿里巴巴开发的分布式服务框架，支持高性能的RPC调用和服务治理，提供了灵活的配置和扩展机制，主要用于Java应用。
  4. Thrift RPC： 是Apache Thrift框架的一部分，专注于提供跨语言的RPC支持，支持多种编程语言和多种传输协议。
  5. RESTful API： 虽然不是传统意义上的RPC框架，但RESTful API也是一种常用的远程调用方式，通过HTTP协议进行通信，使用标准的HTTP方法（如GET、POST、PUT、DELETE）来操作资源。
  6. Spring Cloud： 是一个基于Spring Boot的分布式系统开发工具，提供了丰富的微服务组件和通信框架，包括RPC调用、服务注册与发现、负载均衡等功能。
  7. ZeroMQ： 是一个高性能的消息传递库，支持多种消息传输模式，包括请求-应答模式，可以用于构建简单的RPC系统。
  8. RPyC（Remote Python Call）： 是一个轻量级的Python RPC框架，支持在Python中实现远程过程调用。

像这种远程服务调用，之前了解学习过Feign，那么Feign是RPC框架吗？

Feign 是一个声明式的 HTTP 客户端框架，通常与 Spring Cloud 等微服务框架一起使用。虽然 Feign 本身并不是严格意义上的 RPC 框架，但它可以被用于实现远程服务调用，因此在某种程度上也可以被视为一种 RPC 的实现方式。

Feign 的主要作用是简化服务间的 HTTP 调用，通过定义接口和注解的方式，让开发者可以像调用本地方法一样调用远程服务，而无需手动编写 HTTP 请求和处理代码。Feign 还支持负载均衡、服务发现等功能，使得服务调用更加方便和灵活。

虽然 Feign 在实现上更倾向于基于 HTTP 的服务调用，而不是传统的 RPC 调用，但在微服务架构中，它经常被用来实现服务间的远程调用，因此可以被看作是一种轻量级的 RPC 框架。

RPC的调用流程

  1. 客户端也就是服务消费方以本地方式调用服务；
  2. Client Stub（服务端本地的）接受到调用后负责把方法和参数组装成能够进行网络传输的消息体；
  3. 接下来Client Stub需要在其找到该服务的地址，然后将刚刚组装的消息体发送给这个服务；
  4. 同步的Server Stub收到刚刚发送来的这个消息，然后进行解码；
  5. Server Stub根据编码结果安排其本地的服务调用；
  6. 本地的Server开始执行，得到结果后将这个结果返回给Server Stub；
  7. 同样的这时候的Server Stub也跟Client Stub一样，将这个结果消息组装然后发送给刚刚发来的消费端；
  8. Client Stub进行解码；
  9. 服务端得到最后的结果。

RPC就是将这些内部的操作进行封装，让用户无需关注这些细节。

gRPC

前言

gRPC 是一个高性能、通用的开源 RPC 框架，其由 Google 2015 年主要面向移动应用开发并基于 HTTP/2 协议标准而设计，基于 ProtoBuf 序列化协议开发，且支持众多开发语言。

gRPC支持跨语言使用、基于IDL文件定义服务、通信协议基于标准的HTTP/2设计、序列化支持PB

GRPC是RPC框架中的一种，是一个高性能，开源和通用的RPC框架，基于Protobuf序列化协议开发，且支持众多开发语言。

面向服务端和协议端，基于http/2设计，带来诸如双向流，流控，头部压缩，单TCP连接上的多路复用请求等特性。这些特性使得其在移动设备上表现的更好，更省电和节省空间。

在GRPC里客户端可以向调用本地对象一样直接调用另一台不同机器上服务端医用的方法，使得您能够更容易地创建分布式应用和服务。

与许多RPC系统类似，GRPC也是基于以下理念：定义一个服务，指定其能够被远程调用的方法。在服务端实现这个接口。并运行一个GRPC服务器来处理客户端调用。在客户端拥有一个存根能够向服务端一样的方法。

GRPC特性以及应用场景

（1）特性：

grpc可以跨语言使用。支持多种语言 支持C++、Java、Go、Python、Ruby、C#、Node.js、Android Java、Objective-C、PHP等编程语言。

基于 IDL ( 接口定义语言)文件定义服务，通过 proto3 工具生成指定语言的数据结构、服务端接口以及客户端 Stub。

通信协议基于标准的 HTTP/2 设计，支持双向流、消息头压缩、单 TCP 的多路复用、服务端推送等特性，这些特性使得 gRPC 在移动端设备上更加省电和节省网络流量。

序列化支持 PB（Protocol Buffer）和 JSON，PB 是一种语言无关的高性能序列化框架，基于 HTTP/2 + PB, 保障了 RPC 调用的高性能。

  • 安装简单，扩展方便（用该框架每秒可达到百万个RPC）

（2）使用场景：

  • 微服务：gRPC 设计用于低延迟和高吞吐量通信。 gRPC 对于效率至关重要的轻量级微服务非常有用。

  • 点对点实时通信：gRPC 对双向流式传输提供出色的支持。 gRPC 服务可以实时推送消息而无需轮询。

  • 多语言环境：gRPC 工具支持所有常用的开发语言，因此，gRPC 是多语言环境的理想选择。

  • 网络受限环境：gRPC 消息使用 Protobuf（一种轻量级消息格式）进行序列化。 gRPC 消息始终小于等效的 JSON 消息。

Protocol Buffers

Protocol Buffers（protobuf）是一种轻量级、高效的数据交换格式和编程接口，由 Google 开发。它可以用于结构化数据的序列化，使得数据在网络上传输和存储时更加高效。同时ProtoBuf 是一种更加灵活、高效的数据格式，与 XML、JSON 类似，在一些高性能且对响应速度有要求的数据传输场景非常适用。

ProtoBuf 在 gRPC 的框架中主要有三个作用：定义数据结构、定义服务接口，通过序列化和反序列化方式提升传输效率。

那为什么会提高传输效率呢？这是因为他会将这个ProtoBuf先进行序列化，然后再进行传输，即传输的是二进制数据。

其传输的性能比JSON会快很多，即传输效率会高很多。这也就是为什么gRPC比Feign调用快的原因

Feign和gRPC的区别

Feign就像是一个简单直观的便签系统，适合快速交流和适应性强，但可能在效率和精确性上不如gRPC。而gRPC则像是高效的无线电通讯系统，适合需要快速、准确和跨部门协作的场景。选择哪种沟通方式，取决于餐厅的具体需求和环境。如果你的餐厅需要快速响应顾客需求，并且注重沟通的效率和准确性，gRPC可能是更好的选择。如果你的餐厅更注重简单易用和快速上手，Feign可能更适合你。

  • gRPC：适合高性能、低延迟、跨语言的服务调用，但可能需要额外的配置和集成。
  • Feign：适合Java生态系统，简化了HTTP客户端的开发，易于集成服务发现和容错机制。

简介

Dubbo是阿里巴巴开源的基于 Java 的高性能 RPC（一种远程调用） 分布式服务框架，致力于提供高性能和透明化的RPC远程服务调用方案，以及SOA服务治理方案。

每天为2千多个服务提供大于30亿次访问量支持，并被广泛应用于阿里巴巴集团的各成员站点以及别的公司的业务中。

简单的说，Dubbo就是个服务框架，如果没有分布式的需求，其实是不需要用的，只有在分布式的时候，才有Dubbo这样的分布式服务框架的需求。

并且本质上是个远程服务调用的分布式框架（告别Web Service模式中的WSdl，以服务者与消费者的方式在Dubbo上注册）

  • Dubbo是阿里巴巴公司开源的一个高性能、轻量级的 Java RPC 框架。
  • 致力于提供高性能和透明化的 RPC 远程服务调用方案，以及 SOA 服务治理方案。
  • 官网：http://dubbo.apache.org
