Nginx简介

Nginx 同 Apache 一样都是一种 Web 服务器。基于 REST 架构风格，以统一资源描述符（Uniform Resources Identifier）URI 或者 统一资源定位符（Uniform Resources Locator）URL 作为沟通依据，通过 HTTP 协议 提供各种网络服务。

Nginx是一款 轻量级 的 Web服务器、反向代理服务器，由于它的内存占用少，启动极快，高并发能力强，在互联网项目中广泛应用。

Nginx是一个高性能、灵活和可扩展的Web服务器和代理服务器，适用于各种场景，包括静态文件服务、反向代理、负载均衡和动态内容处理等。

正向代理&反向代理

通过图可以明显的看出他们两者的区别。

正向代理就是代理的用户，而且用户是知道访问目标是哪个的。

反向代理代理的是服务端，这就是反向的意思。同时用户的请求会被哪个处理是不清楚的，这部分对于客户端来说是透明的。

在智慧加油站项目中使用了反向代理，为什么需要进行反向代理呢？

隐藏真实服务器：通过反向代理，可以隐藏真实的服务器架构和拓扑结构，使得客户端无法直接访问到后端服务器。这提高了服务器的安全性，防止了直接暴露服务器的 IP 地址和内部结构。

缓存静态内容：反向代理可以缓存静态内容，例如图片、CSS、JavaScript 等，从而减轻后端服务器的负担，并提高网站的访问速度和性能。

负载均衡

负载均衡（Load Balance）其意思就是分摊到多个操作单元上进行执行，例如Web服务器、FTP服务器、企业关键应用服务器和其它关键任务服务器等，从而共同完成工作任务。

简单来说就是：现有的请求使服务器压力太大无法承受，所有我们需要搭建一个服务器集群，去分担原先一个服务器所承受的压力，那现在我们有ABCD等等多台服务器，我们需要把请求分给这些服务器，但是服务器可能大小也有自己的不同，所以怎么分？如何分配更好？又是一个问题。

Nginx关于负载均衡的方式有三种实现方式

  1. 轮询法

顾名思义，按照时间顺序，轮到谁了就把这个请求交给谁去处理

  1. 加权轮询

每个服务器接收不同数量的请求，即分配权重。但是这样会出现请求访问其他服务器时，没有该用户登录信息的问题。

  1. ip_hash

我们可以采用 ip_hash 指令解决这个问题，如果客户已经访问了某个服务器，当用户再次访问时，会将该请求通过哈希算法，自动定位到该服务器。每个请求按访问 ip 的 hash 结果分配，这样每个访客固定访问一个后端服务器，可以解决 session 的问题。

动静分离

动静分离，在我们的软件开发中，有些请求是需要后台处理的，有些请求是不需要经过后台处理的（如：css、html、jpg、 js 等文件），这些不需要经过后台处理的文件称为 静态文件。让动态网站里的动态网页根据一定规则把不变的资源和经常变的资源区分开，动态资源做好了拆分以后，我们就可以根据 静态资源 的特点将其做缓存操作，以提高资源的响应速度。

当用户请求目标是静态数据时，这时候就不需要让后端对这个请求处理了，因为静态数据是不变的，所以就可以让Nginx直接处理，减低服务端的压力。

Nginx主配置文件

可以对比看智慧加油站项目中的Nginx的主配置文件。这里截取了一段，关键的内容。

proxy_pass http://192.168.0.29:59999;这里将动态请求交给ip为192.168.0.25端口号59999的服务进行处理。

IP是我本机的ip地址，如果需要在自己的电脑上运行，需要在这里把ip地址改成你自己的ip地址。端口号是智慧加油站网关微服务的端口号，所有的动态请求一律交给网关进行处理。

如果把网站看成一个公司，现在你招了一个Nginx的员工，当他入职的那天，他会给你几张纸条（配置文件）。请你在上面写上他的工作内容。即他需要监听哪些请求（IP：端口号），这些端口号分配给哪些服务（IP：端口号）。我们只需要在配置文件中进行指定即可。
