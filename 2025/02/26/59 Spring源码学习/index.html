@SpringBootApplication

该注解是一个组合注解，其注解是由其它的注解组合起来从而得到的，他的作用就是标记和说明这个类是Springboot的主配置类。

该注解主要由下面三个注解组成：

  1. @SpringBootConfiguration
    @SpringBootConfiguration = @Configuration + @Component

    表示这个类是一个配置类，而配置类也是Spring容器中的组件

    该注解主要标记启动类，并为Springboot应用提供额外的配置支持

  2. @EnableAutoConfiguration

    @EnableConfiguration = @AutoConfigurationPackage + @Import

    @EnableAutoConfiguration 是 Spring Boot 的核心注解之一。它的作用是根据应用的类路径、项目所声明的 Bean 定义以及其他环境配置自动配置 Spring 应用的上下文。

    @AutoConfigurationPackage该注解就是将该启动类所在包以及其子包中的所有组件都扫描并加载到Spring的容器中，这也就是为什么我们在利用springboot进行开发的时候，无论是Controller还是Service的路径都是与主配置类同级或者次级的原因。

  3. @ComponentScan

    该注解用于自动扫描指定包及子包中的Spring组件，这里和上面不同之处在于，该注解支持指定包，他会将指定包及其子包中所有的Spring组件都注册成Spring容器中的Beans。

Spring IoC

参考资料：【Spring成神之路】一次SpringIOC源码之旅，女朋友也成为了Spring大神！

前言

从Spring IOC的使用过程中，我们可以非常直观的感受到Spring IOC代替我们程序员完成了一个对象的创建和管理过程。

也许你觉得不就是new 一个对象吗，何必需要专门设计一个IOC来替我们管理呢？这样不是反而会更复杂吗？也许我们可能想错了，如果在我们的程序中越来越复杂，一个对象的依赖关系会越来越复杂，比如类A同时依赖类B、类C、类D，而类B又依赖类X、类Y、类Z，当你尝试new一个这样的对象不可避免的会感觉心累。

前言
