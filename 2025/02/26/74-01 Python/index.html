74-01 Python

结合你对Java的熟悉程度，我会详细介绍Python中的面向对象编程（OOP）语法，并与Java进行对比。Python和Java的OOP概念相似，但语法和一些实现细节有所不同。

面向对象

1. 类与对象

对比：

  • 类定义：Python使用class关键字定义类，类似于Java。
  • 构造方法：Python用__init__方法实现构造函数，而Java使用与类名相同的方法。
  • self关键字：Python中的self类似于Java中的this，表示类的实例。
  • 方法定义：Python的方法必须显式包含self作为第一个参数，用于访问实例变量；Java则隐式传递this。

  • 方法重写：Java使用@Override注解表示重写方法，而Python不需要特殊标注，直接定义同名方法即可。

数据结构

在Python中，数据结构是处理和存储数据的基础工具。Python提供了多种内置的数据结构，涵盖了从基本类型到复杂类型的各种场景需求。基于我对Python的理解，以下是对主要数据结构的详细介绍：

1. 列表（List）

Python的list是动态数组，类似于Java中的ArrayList，可以存储任何类型的元素，并且大小可变。

基本操作：

特点：

  • 有序：元素按照插入顺序存储，支持索引访问。
  • 动态大小：支持动态添加、删除元素。
  • 支持多种类型：可以存储不同类型的元素在同一列表中。

2. 元组（Tuple）

tuple类似于list，但不可变，这意味着一旦创建，元素就不能修改。

基本操作：

特点：

  • 不可变：提高了安全性，适合存储固定数据。
  • 性能优越：由于不可变，元组操作比列表更快。

3. 字典（Dictionary）

Python的dict类似于Java中的HashMap，存储键值对，提供了快速的查找和插入操作。

基本操作：

特点：

  • 无序（Python 3.6之前）：Python 3.7及以后的版本中，字典保持插入顺序。
  • 键唯一：每个键在字典中唯一，值可以是任何类型。

4. 集合（Set）

set类似于Java中的HashSet，用于存储不重复的元素，支持数学集合运算。

基本操作：

特点：

  • 无序：集合中的元素没有固定顺序。
  • 不重复：集合中的每个元素是唯一的。
  • 高效查找：集合的查找和插入操作时间复杂度为O(1)。

5. 队列（Queue）

Python提供collections.deque作为双端队列（deque），可以在两端高效地添加和删除元素。类似于Java中的LinkedList实现的队列。

基本操作：

特点：

  • 双端队列：可以从两端高效地添加和删除元素。
  • 线程安全：deque的操作是原子性的，适合多线程场景。

6. 栈（Stack）

Python没有专门的栈结构，但可以使用list模拟，或者使用collections.deque实现更高效的栈操作。

使用list作为栈：

特点：

  • 后进先出（LIFO）：栈的操作遵循后进先出原则。

7. 优先队列（Priority Queue）

Python提供heapq模块来实现优先队列，底层基于堆结构。

基本操作：

特点：

  • 最小堆：heapq默认实现最小堆，heappop返回最小元素。
  • 时间复杂度：插入和删除操作的时间复杂度为O(log n)。

8. 字符串（String）

Python中的str是不可变对象，类似于Java的String，用于存储字符序列。

基本操作：

特点：

  • 不可变：字符串一旦创建，无法修改。
  • 丰富的操作方法：Python提供了大量字符串操作方法，如split()、join()、replace()等。

9. 冻结集合（Frozenset）

frozenset是不可变的集合，类似于set，但元素不可变，适合用于作为字典的键或其他需要不可变性的场景。

基本操作：

特点：

  • 不可变：创建后不可修改。

10. 字节（Bytes）

bytes用于处理二进制数据，类似于Java中的byte[]。

基本操作：

特点：

  • 不可变：bytes对象是不可变的。
  • 用于二进制数据处理：常用于网络通信、文件I/O等场景。
