简介

Spring Security是Spring框架中的一个安全框架，它致力于应用的两个方面：认证和授权。简而言之就是验证你是谁？和你是否有权限访问这个资源（角色）。

它是一个强大的、高度自定义的认证和访问控制框架。其核心就是一组过滤器链，项目启动后会自动配置。

在接触Spring Security之前我们会使用一组过滤器过滤掉无登录状态的请求和一些非法请求，而Spring Security就是将这些功能封装好了，无需再重复书写。

其架构图如上所示，在这些过滤器中最关键的就是Basic Authentication Filter，该过滤器用来验证当前用户的身份也就是角色，当前用户具有什么权限。

思路

创建一个新工程时，我们需要思考一下我们接下来需要的一些步骤，需要做什么，怎么做。

  • 搭建springboot工程
  • 导入springSecurity跟jwt的依赖
  • 用户的实体类
  • dao层
  • service层（真正开发时再写，这里就直接调用dao层操作数据库）
  • 实现UserDetailsService接口
  • 实现UserDetails接口
  • 验证用户登录信息的拦截器
  • 验证用户权限的拦截器
  • springSecurity配置
  • 认证的Controller以及测试的controller
  • 测试

配置拦截器

在spring-security中需要实现两个过滤器，一是用户验证，第二是鉴权。。使用JWTAuthenticationFilter去进行用户账号的验证，使用JWTAuthorizationFilter去进行用户权限的验证。
