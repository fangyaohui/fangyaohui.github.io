Maven是什么

Maven 是 Apache 软件基金会组织维护的一款专门为 Java 项目提供构建和依赖管理支持的工具。

一个 Maven 工程有约定的目录结构，约定的目录结构对于 Maven 实现自动化构建而言是必不可少的一环，就拿自动编译来说，Maven 必须 能找到 Java 源文件，下一步才能编译，而编译之后也必须有一个准确的位置保持编译得到的字节码文件。

如果在开发中如果需要让第三方的工具或者框架知道我们自己创建的资源在哪，那么基本上就是两种方式：

  1. 通过配置的形式明确告诉它
  2. 基于第三方工具或框架的约定 Maven 对工程目录结构的要求

构建环节

依赖

在Maven中，依赖（Dependencies）是指项目所依赖的外部库、模块或者其他项目。简单来说，当你的项目需要引用其他代码库或者模块提供的功能时，你可以将这些外部资源声明为你项目的依赖。依赖管理是Maven的一个核心功能，它帮助你轻松地管理项目所需的各种外部资源。

Maven使用一种叫做坐标（Coordinates）的方式来唯一标识每个依赖项。这个坐标通常包括组织（Group）、名称（Artifact）和版本（Version）三个部分。组织标识了库的所属组织或者项目，名称则标识了具体的库或者项目名称，版本则指明了库或者项目的具体版本号。

就好比食谱，上面记录着你这菜需要的所有原材料，材料是什么，来自哪里的，需要什么时期的，都会记录在上面。

在有些时候A依赖B，B依赖C，而且C可以传递到A的时候，但是A不想要C，这时候就想把C排除在外面。

本地仓库

下载好Maven后，我们需要在本地创建一个文件夹，用来存放我们下载到本地的所有菜单。然后我们需要在setting文件中指定本地仓库的位置，以便让Mavne可以找到本地仓库。这样Maven就会将下载好的依赖存放在本地仓库中同时需要依赖的时候也会从本地仓库中寻找。

这里我们需要重点介绍一下刚刚提到的maven的setting文件，如果我们找到setting文件，可以发现其文件类型是xml，即为配置文件。那么该配置文件会为maven配置一些参数，比如本地仓库的地址。

Maven的使用

之前我们介绍了到了，maven会根据菜单（pom）文件去菜市场（中央仓库）找到需要的东西。但是中央仓库很多，东西很多，我们该如何找到我们最需要的那个物品呢？

这里就引入了坐标的概念，世界是三维的，那么我们就可以根据三个坐标点来确定一个具体位置。那么对于maven来说也是一样的，只不过在maven世界里不是xyz而是groupId、artfactId和version。

  1. groupId：公司或组织的id，例如：com.javatv.maven；
  2. artfactId：一个项目或者是项目中的一个模块的 id，即模块的名称，将来作为 Maven 工程的工程名；
  3. version：版本号。

继承

Maven工程直接存在继承的关系，比如A工程继承B工程，那么B工程就是父工程，A工程就是子工程。但是这样有什么用？

dependencies 与 dependencyManagement 的区别

  1. 父模块 中定义的所有依赖，在子项目中都会直接继承；
  2. 在父模块 中定义的所有依赖，子项目并不会继承，我们还要在子项目中引入我们需要的依赖，才能进行使用。此时我们在子项目中不用设置版本。

注意：

（1）子模块继承的父模块的依赖/子模块声明的依赖，如果在子模块中没有被使用，在子模块单独打包时，就不会被打入 jar ；

（2）maven 打包加入的依赖代码是被调用到的部分代码，没有被调用到的依赖代码不会被加入打包后的 jar 包中。
