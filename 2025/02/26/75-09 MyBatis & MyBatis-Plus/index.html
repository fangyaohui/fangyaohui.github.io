整体架构

接口层

指用于操作数据库的增删改查接口，如SqlSession以及用户自定义的Mapper接口，都属于接口层组件。通常我们对MyBatis源码学习是从接口层开始的，但这并不是一个好方法，至于原因会在学习路径中讲到。

数据处理层

SQL执行流程实现，内容包括动态SQL语句构建、参数映射处理、结果映射集处理。他是整个MyBatis源码的核心，也是我们学习的重点和难点。

扩展与支撑层

指对主流程功能上的扩展与支撑。

  • 扩展：对主流程，在易用性、性能上进行扩展。如SqlSession与Mappr接口就是易用性扩展。缓存与懒加载则是执行性能的扩展。
  • 支撑：主程所依赖的工具，如数据源、反射工具、日志管理、配置构建器、脚本执行器等学习重点

数据处理层是MyBatis核心与重点，我们学MyBatis源码其实就是学它，搞定它MyBatis源码就掌握了80%，当然它的难度也是最大的。为方便理解，我把它分成了执行流程与映射配置两部分。

  • 执行流程：体现SQL执行过程中的每个步骤。
  • 映射配置：SQL执行前后的动态SQL解析、参数映射、结果集映射。

学习目标分解

第一目标-Sql执行流程

MyBatis源码学习的第一个目标就是要搞定四个组件所组成的Sql执行流程。这个四个组件分别是，执行器(Execute)、Sql处理器(StatementHandler)、参数处理器(ParameterHandler)、结果集处理器(ResultSetHandler)。这个目标又可分解成4个小目标

  1. 理解四个组件执行上的顺序
  2. 每个组件的作用及意义
  3. 掌握Execute 的三个实现逻辑
  4. 掌握StatementHandler的三个实现逻辑

这个目标不是最难的，但是最要的。如果有一天你把学的MyBatis源码忘的差不多了，也希望你这个流程不要忘，这是MySql源码的主干和肪脉络。顺着它就可以把MyBatis知识再拾回来。

说明：关于参数处理、结果集处理其主要逻辑细节在于映射处理，可以放到映射篇章学习。

第二目标：映射配置

第二个要搞定的目标，就是SQL执行流程当中，所需要的三大映射：分别是动态SQL映射、参数映射、以及结果集映射。不仅要搞清楚映射配置的加载与解析，更重要的弄清楚映射逻辑。

  • 加载解析：指用户配置的XML解析成JAVA配置对象。比如块,最终要解析成SqlSource。或… 要解析成ResultMap对象。
  • 映射逻辑：比如SqlSource如何基于参数生成可执行SQL，ResultMap如何将结果集，解析成JAVA对象等。

这里面涉及内容非常多，同样可以划分成一个个小目标：

小目标
1 理解Sql映射声明(MappedStatement)的作用，以及它与三大映射的关系。
2 掌握动态SQL(sqlSource)解析过程，包括其脚本表达示的执行逻辑
3 掌握MyBatis所封装的映射工具包的使用
4 掌握参数转换过程，以及映射的封装过程
5 掌握结果集映射核心逻辑，包括关联映射，1对1、1对多的处理。还有更复杂的循环依赖处理
6 掌握全部配置(Confgiuration) 的核心定位,以及它的加载解析过程

数据处理层

这一层所涉及到的知识点就是MyBatis的核心知识点，如果把这部分拿下的，MyBatis以及拿下七八十了。

MyBatis重点就是上面的四大核心组件：执行器、SQL处理器、参数处理器和结果集处理器。我们需要重点关注其中这四个组件代表的具体含义是什么？这四个组件的执行顺序是什么？以及他们是如何实现的？

从上面的这个图中我们可以明显的看出这四个组件的执行顺序是：执行器、SQL处理器、参数处理器然后就是结果处理器。但是这四个组件的职责是什么？以及如何实现的呢？

在详细的介绍MyBatis中的SQL执行流程之前，我们先回顾一下JDBC中的SQL执行流程：

  1. 我们需要使用Class.forName()方法加载数据库驱动类
  2. 获取到连接对象，其底层的实现是基于Socket进行数据库连接的
  3. 执行SQL语句，放回执行结果
  4. 处理执行结果，并与结果集进行映射，然后释放资源

执行器

Executor是MyBatis的核心组件，管理整个SQL的执行流程。可以理解成是MyBatis中的“大管家”，负责调用其他组件并管理缓存、事务等。Executor有三种实现方式：

  • SimpleExecutor：最简单的执行器，每次都会创建新的Statement进行SQL执行。这种方式不使用缓存，适合简单的查询需求。
  • ReuseExecutor：复用已有的Statement对象，适用于同一连接中反复执行相似SQL的场景，减少Statement的创建次数。
  • BatchExecutor：批处理执行器，将多条SQL语句组合成一个批处理，减少网络往返。适用于大量相似的插入、更新操作。

流程简析：Executor的工作流程从接收到SQL请求开始，到数据库获取结果并返回给调用方结束。在流程中，Executor会选择具体的StatementHandler去处理SQL的生成和执行。

Executor是MyBatis核心，是整个SQL执行的入口，它负责管理和协调其它组件的工作，其主要是发起、执行SQL语句并在执行的过程中进行必要的管理、缓存和事务控制。

执行器也提供了缓存机制，其提供了一级缓存和二级缓存，提高了查询效率，降低了对数据库的访问。

  1. 一级缓存：针对的是同一个SqlSession在一定时间内执行相同的SQL查询时会被缓存，从而避免多次访问数据库；
  2. 二级缓存：针对的是mapper层面，在不同的SqlSession中，在一定时间内执行相同的SQL查询会被缓存，从而避免多次访问数据库。

同时该执行器也提供了事务支持的能力，执行器可以根据需求控制事务的提交和回滚，实现事务的原子性

MyBatis注解

@Intercepts & @Signature

参考资料
  1. Mybatis拦截器注解@Intercepts与@Signature注解属性说明
前言

@Intercepts 是 MyBatis 提供的注解，用于定义 MyBatis 拦截器（Interceptor）要拦截的方法。MyBatis 拦截器是一个强大的机制，允许你在方法执行前后添加额外的处理逻辑，比如修改 SQL 语句、记录日志、事务处理等。

@Intercepts可以拦截以下四种类型：

  • Executor sql的内部执行器
  • ParameterHandler 拦截参数的处理
  • StatementHandler 拦截sql的构建
  • ResultSetHandler 拦截结果的处理

@Signature 注解用于定义具体的拦截点，包括：

  • type：就是指定拦截器类型（ParameterHandler ，StatementHandler，ResultSetHandler ）
  • method：是拦截器类型中的方法，不是自己写的方法
  • args：是method中方法的入参

@Intercepts 和 @Signature 是 MyBatis 的注解，用于定义拦截器的元信息，指定拦截的目标类型、方法和参数列表。这四个注解分别拦截 MyBatis 执行的四种核心组件（Executor、StatementHandler、ParameterHandler 和 ResultSetHandler），通过它们可以在不同阶段插入自定义逻辑。

1. @Intercepts

@Intercepts 是一个容器注解，表示一个拦截器可以拦截多个方法。它包含一个 @Signature 数组，每个 @Signature 注解定义了一个拦截目标。

关键点：

  • 作用：标注在拦截器类上，声明这个类是 MyBatis 的一个拦截器。
  • 参数：包含多个 @Signature 注解，每个注解定义了一个具体的拦截点。

2. @Signature

@Signature 是定义拦截点的核心注解，它指定了：

  1. 拦截的类型（type）
    指定要拦截的 MyBatis 核心组件接口，例如 Executor、StatementHandler 等。
  2. 拦截的方法（method）
    指定要拦截的方法名称，例如 query、prepare 等。
  3. 拦截方法的参数（args）
    指定方法参数类型的完整签名，用于精确定位目标方法。

详细说明：

  • type：指定目标接口。

四个 @Signature 注解的详细介绍

1. 拦截 Executor.query 方法

  • 作用：拦截执行查询操作的核心方法。

  • 目标方法

    ：

    • Executor.query(MappedStatement ms, Object parameter, RowBounds rowBounds, ResultHandler resultHandler)

  • 常见用途

    ：

    1. 修改 SQL 查询语句。
    2. 添加动态查询条件。
    3. 实现分页逻辑（如自定义分页插件）。

2. 拦截 StatementHandler.prepare 方法

  • 作用：拦截 SQL 语句的预处理阶段。

  • 目标方法

    ：

    • StatementHandler.prepare(Connection connection, Integer transactionTimeout)

  • 常见用途

    ：

    1. 动态修改 SQL。
    2. 打印完整的 SQL 语句（包括参数）。
    3. 添加审计日志记录。

3. 拦截 ParameterHandler.setParameters 方法

  • 作用：拦截设置 SQL 参数的阶段。

  • 目标方法

    ：

    • ParameterHandler.setParameters(PreparedStatement ps)

  • 常见用途

    ：

    1. 动态加密/解密字段参数。
    2. 自定义参数值的绑定逻辑。
    3. 检查或修改绑定参数。

4. 拦截 ResultSetHandler.handleResultSets 方法

  • 作用：拦截结果集的处理阶段。

  • 目标方法

    ：

    • ResultSetHandler.handleResultSets(Statement stmt)

  • 常见用途

    ：

    1. 对查询结果进行加密/解密。
    2. 自定义结果集映射逻辑。
    3. 过滤或调整查询结果。

拦截器的工作机制

  1. 动态代理
    MyBatis 使用动态代理模式，将核心组件（如 Executor）包装成代理对象。
  2. 执行拦截逻辑
    在拦截的方法前后，执行 Interceptor 实现类的 intercept 方法。
  3. 传递调用链
    通过 invocation.proceed() 调用被代理对象的原始方法。

应用场景总结

  • Executor.query：用于动态 SQL、日志记录、分页逻辑增强。
  • StatementHandler.prepare：用于动态修改 SQL 语句、审计。
  • ParameterHandler.setParameters：用于参数加密、绑定自定义逻辑。
  • ResultSetHandler.handleResultSets：用于结果解密、数据过滤。

通过灵活使用拦截器，可以对 MyBatis 的行为进行深度定制，从而实现高级功能和扩展。
